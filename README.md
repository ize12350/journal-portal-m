# journal-portal-m

## 项目简介

`journal-portal-m` 是一个用于归档与发布多个独立 HTML 页面的仓库。  
本项目不绑定任何特定域名或网站，仅作为一组静态 HTML 页面的集中存放与版本管理工具。  
所有页面内容均为独立归档，便于后续查阅、分享或重新部署。

## 目录说明

```
journal-portal-m/
├── pages/          # 存放所有 HTML 页面文件
├── assets/         # 页面所需的静态资源（CSS、JS、图片等）
├── index.html      # 入口页面（可选）
└── README.md       # 本文件
```

- `pages/`：每个 HTML 文件独立存放，文件命名尽可能体现内容或用途。
- `assets/`：按需组织，资源文件与页面文件分离，保持结构清晰。
- `index.html`：如存在，作为仓库预览时的默认入口，非必需。

## 页面归档说明

本仓库收录的 HTML 页面均为独立归档，遵循以下原则：

- 每个页面是自包含的 HTML 文件，依赖的外部资源统一存放于 `assets/` 目录。
- 页面不嵌入特定域名或第三方跟踪代码，保持纯粹的文档/工具属性。
- 归档目的仅为保存与展示，不主动推广或诱导访问。

## 维护说明

- 新增页面：将 HTML 文件放入 `pages/` 目录，并在 `index.html`（若有）中添加链接。
- 更新页面：直接修改对应文件，建议在 commit 信息中说明变更内容。
- 删除页面：移除文件及 `index.html` 中的对应链接。
- 资源管理：如需更新公共资源，请确保不影响其他页面的正常使用。
